# 自主工程循环

> Plan → Execute → Verify → Fix。不是代码生成器，是自主工程师。

## 循环流程

```
用户任务
    │
    ▼
┌─ 规划 (PLAN) ───────────────────────┐
│ 1. todo_write — 分解任务             │
│ 2. read_file / grep / glob — 探索    │
│ 3. find_code — 语义搜索              │
└──────────────┬───────────────────────┘
               ▼
┌─ 执行 (EXECUTE) — TDD优先 ──────────┐
│ 4. 编写测试 (定义行为)               │
│ 5. 运行测试 (应该失败)               │
│ 6. 编写代码 (最小通过)               │
│ 7. git stash create ← 安全网         │
└──────────────┬───────────────────────┘
               ▼
┌─ 验证 (VERIFY) — 强制执行 ──────────┐
│ 8. detect_verify → 查找验证命令      │
│ 9. run_verify → 一键运行全部         │
└──────────────┬───────────────────────┘
               │
       ┌───────┴───────┐
       ✅ 全部通过       ❌ 失败
       │               │
       ▼               ▼
┌─ 完成门控 ──  ┌─ 修复 (max 3轮) ────────┐
│ 5项全过=done  │ 分析错误 → 修复 → 重验   │
└───────────────┘ └──────────────────────────┘
```

## 五维完成门控

"完成"不是 Agent 说了算。五个门控必须**全部通过**：

| 门控 | 检查 | 阻塞条件 |
|---|---|---|
| **测试** | 测试套件退出码 0 | 任何测试失败 |
| **类型检查** | `tsc --noEmit` / `mypy` 无错误 | 存在类型错误 |
| **调试残留** | 无 `console.log` / `print` / `debugger` | 遗留调试语句 |
| **Diff范围** | `git diff --stat` 变更聚焦 | > 500行插入 |
| **已验证** | 至少运行过一次验证命令 | 写了代码但从未测试 |

## 三大强制机制

### 验证阻塞

Agent 写了代码但试图在没跑测试的情况下说"done"：

> `[SYSTEM] 你修改了代码但尚未验证。立即运行项目的测试套件和类型检查。`

`hasVerified` 标志位在每次代码编辑时自动重置为 `false`，只有 `run_verify` 成功后才设为 `true`。

### 失败反思注入

测试失败时自动注入结构化提示。`verifyNotDoneCount` 限制验证循环最多 2 次阻塞，防止死循环。

### 重试上限

最多 3 次修复尝试（75% 的改进来自前 2 轮）。3 轮后诚实报告剩余错误。

## ReAct 反思推理

每 5 步自动触发一次反思：

```
[步骤 5/50] ReAct 反思:
  - 当前进展: 已完成3个子任务，剩余2个
  - 下一步: 继续处理认证模块
  - 是否偏离计划: 否
```

## 声明式验证 `run_verify`

`run_verify` 工具自动检测并执行项目所有验证命令：

```
run_verify()
  → 检测: pnpm test / vitest run
  → 检测: tsc --noEmit
  → 检测: eslint src/
  → 检测: pnpm build
  → 全部执行，汇总结果
```

支持 25+ 种语言/框架的自动检测。
