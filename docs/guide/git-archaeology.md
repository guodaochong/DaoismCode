# 🔍 Git Archaeology `/why`

> **代码考古** — 从 git 历史中还原代码的设计意图、演进过程和风险信号。

## 问题

你接手了一个项目，看到一段代码，不明白：
- 为什么这样写？当初解决什么问题？
- 谁写的？改了多少次？稳定还是频繁变更？
- 有没有被回退过？是否存在争议？

传统的 `git blame` 只告诉你「谁在什么时候写了这一行」，但不告诉你**为什么**。

## 解决方案

`/why` 命令执行完整的代码考古分析：

```
/why src/auth/login.ts:handleLogin
/why src/utils/parser.ts
```

### 工作流程

```
git blame --line-porcelain ──▶ 逐行状态机解析
         │
         ├─▶ 作者分布 (谁拥有这段代码)
         ├─▶ 热点提交 (哪些 commit 触碰最多行)
         ├─▶ 稳定性分析 (🔥 不稳定 / ✅ 稳定)
         └─▶ 回退检测 (Revert / 回退 / 撤销)
                    │
                    ▼
         git show (提取关键 diff)
                    │
                    ▼
         LLM 生成中文考古报告
```

### 输出示例

```markdown
📍 代码考古: src/auth/login.ts:handleLogin

🔥 稳定性: 高频变更 (不稳定)
⚠️ 争议: 检测到 2 次回退提交

🗓️ 起源: 2024-01-15, 张三在 commit a1b2c3d 引入
  - 解决初始登录流程的安全性认证问题

🔧 演进: 经历 23 次提交修改
  - 热点 commit: d4e5f6a (15行), g7h8i9j (8行)

💡 设计决策: 采用了 JWT + Refresh Token 双令牌策略

⚠️ 风险信号:
  - 代码频繁变更，可能存在设计问题
  - 2 次回退提交，说明存在争议或反复
```

## 功能细节

### 逐行 Blame 解析

使用状态机逐行解析 `git blame --line-porcelain` 输出，提取：
- Commit hash（完整 40 字符）
- 作者姓名
- 提交时间（epoch → ISO 日期）
- 提交摘要

### 热点检测

统计每个 commit 修改的行数，排序取 Top 5。热点 commit 往往是引入重大变更的位置。

### 稳定性分级

| 提交次数 | 级别 | Emoji |
|---|---|---|
| ≥ 15 | 高频变更（不稳定） | 🔥 |
| ≥ 8 | 中等变更 | ⚠️ |
| ≥ 3 | 低频变更（稳定） | ✅ |
| < 3 | 极少变更（非常稳定） | 🧊 |

### 关键 Diff 提取

自动拉取 Top 3 热点 commit 的 `git show` diff（限制 1500 字符），展示实际代码变更。

### LLM 报告生成

将所有原始数据（blame、log、diff、统计）发送给 LLM，生成结构化的中文考古报告。

## 技术规格

- Git 命令超时：30 秒（支持大型仓库）
- 二进制文件检测：自动跳过
- API 失败降级：LLM 分析失败时返回原始数据
- 使用 `git blame -w` 忽略空白变更
- 使用 `git log --follow` 跟踪文件重命名
