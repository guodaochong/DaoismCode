# 四境智能路由

> 六层级联决策引擎，逐消息、逐步骤动态选择最优模型。自动节约 56% 推理成本。

## 核心理念

传统 AI 编码工具用一把锤子敲所有钉子。DaoismCode 的路由器是**动态的**：每条消息、每个执行步骤都独立路由到最优模型。

## 四境体系

路由系统将所有 LLM 按能力划分为四个层级（"境界"）：

| 境界 | 模型类型 | 特点 | 适用场景 | 成本 |
|---|---|---|---|---|
| 🟫 **筑基** (Foundation) | 轻量快速模型 | 高速推理, 低延迟 | 简单问答、文件搜索、代码阅读 | ★☆☆☆☆ |
| 🟨 **金丹** (Golden Core) | 多模态视觉模型 | 图像理解 + 文本推理 | 截图分析、UI转代码、错误诊断 | ★★☆☆☆ |
| 🟦 **元婴** (Nascent Soul) | 平衡推理模型 | 中等推理 + 安全降级 | 日常编码、中等复杂度任务 | ★★★☆☆ |
| 🟪 **化神** (Spirit) | 旗舰编码模型 | 编码SOTA, 超长上下文 | 复杂重构、架构设计、疑难bug | ★★★★★ |

境界的映射是配置化的，不绑定特定厂商。

## 六引擎级联决策

每一层在前一层不确定时介入：

### ① 权重评分引擎

所有类别**同时评分**，取得分最高的：

```
消息: "检查 auth.ts 里的 bug 并优化性能"

评分:
  coding:   1.5    (文件路径 auth.ts +1.5)
  review:   1.8    (bug ×0.9 + optimize ×0.9)
                    ↓
  胜出: review (1.8)  →  化神境
```

纯规则驱动，延迟 <1ms，处理 80%+ 的路由决策。

### ② 工具调用反馈

Agent 上轮用了什么工具，是当前任务类型的最强信号：

| 上轮工具 | 推断类型 | 路由至 |
|---|---|---|
| `edit_file` `write_file` `bash` | 编码 | 化神 |
| `read_file` `glob` `grep` | 搜索 | 筑基 |
| 混合 | 多数投票 | 动态 |

### ③ 上下文感知

记住最近 3 轮对话，解决"简短回复"导致的路由震荡：

```
第1轮: "实现登录功能"        → 编码 → 化神
第2轮: "再加密码重置"         → 编码 → 化神
第3轮: "嗯，还有验证码"       → 模糊... 但最近3轮 = 编码
      → 延续上下文 → 编码 → 化神 ✓
```

### ④ LLM 辅助分类

当规则引擎置信度低时，筑基境执行 0.3 秒快速分类。用最便宜的模型做判断，投入产出比极高。

### ⑤ 复杂度 + 预算守卫

- **复杂度升级**：消息 > 300字 → 自动升级到化神境
- **预算降级**：任一境界 > 85% 配额 → 自动降级

### ⑥ 多步混合路由

整个路由系统的**核心创新**。一个5步任务，逐步切换模型：

```
任务: "搜索登录代码，然后加双因素认证"

  步骤1: glob + grep (搜索)     → 筑基  (便宜，够用)
  步骤2: read_file (读代码)     → 筑基
  步骤3: edit_file (写代码!)    → 自动升级到化神
  步骤4: bash (跑测试)          → 化神
  步骤5: 回复用户               → 化神
```
