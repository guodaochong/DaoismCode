# 增量上下文压缩

> 长对话不再意味着重新摘要一切。DaoismCode 只将新消息合并到现有摘要中——节省 60%+ token，速度快 3 倍。

## 问题

传统的上下文压缩在上下文窗口填满时，会将**所有**旧消息丢给 LLM 做全量摘要。这很慢、很贵，而且每次重新摘要都会损失保真度。

## 解决方案

DaoismCode 的增量压缩方法：

1. **首次压缩** — 对所有旧消息做完整摘要（与之前相同）
2. **后续压缩** — 在上下文中找到已有的 `[Conversation Summary]`，只合并该摘要之后的**新消息**
3. LLM 收到：*已有摘要 + 新消息* → 输出*更新后的摘要*
4. 净效果：只需摘要 3 条消息而不是 50 条，减少 60%+ token 消耗

## Token 对比

```
传统方式: [50条旧消息] → LLM → [摘要]          ~8000 input tokens
增量方式: [上次摘要] + [3条新消息] → LLM        ~1200 input tokens
                                                 ████████████ 85%节省
```

## 技术实现

当 `compactConversation` 被调用时：

1. 扫描消息数组，查找 `[Conversation Summary]` 标记
2. 如果找到已有摘要 → 只压缩该摘要之后的新消息
3. 如果新消息 ≤ 3 条 → 跳过压缩（不值得）
4. LLM 收到 `INCREMENTAL_COMPACT_PROMPT` + 已有摘要 + 新消息
5. 输出合并后的摘要替换旧摘要

## 配置

自动触发，无需配置。当对话超过 50 条消息或 token 使用率超过 50% 时触发。
